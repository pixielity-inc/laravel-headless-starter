# Contributing to Laravel Headless Starter

Thanks for your interest in improving this template! Here's how you can contribute.

## Development Workflow

1. Fork the repository
2. Create a feature branch from `develop`
3. Make your changes
4. Test thoroughly
5. Submit a pull request to `develop` branch

## Branch Strategy

- `main` - Stable releases only
- `develop` - Active development and testing

## Guidelines

- Follow Laravel conventions and best practices
- Keep the template minimal and focused
- Test with a fresh Laravel installation
- Update documentation for any new features
- Run `vendor/bin/pint` before committing

## What to Contribute

- Bug fixes
- Documentation improvements
- New API patterns or examples
- Performance optimizations
- Security enhancements

## What Not to Include

- Frontend frameworks (keep it headless)
- Opinionated packages (keep it minimal)
- Business logic (it's a starter template)

## Questions?

Open an issue for discussion before starting major changes.
